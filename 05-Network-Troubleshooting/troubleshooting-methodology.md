# Network Troubleshooting Methodology

## Purpose of Troubleshooting

Network troubleshooting is the structured process of identifying, diagnosing, and resolving network issues.

Using a consistent methodology prevents guesswork and reduces downtime.

---

## CompTIA Troubleshooting Steps

1. Identify the problem
2. Establish a theory of probable cause
3. Test the theory
4. Establish a plan of action
5. Implement the solution
6. Verify full system functionality
7. Document findings and actions

---

## Applying the Methodology

Example scenario:
User cannot access a website.

Steps applied:
- Identify symptoms
- Test connectivity
- Check DNS resolution
- Verify firewall rules
- Confirm resolution

---

## Security Relevance

SOC analysts use structured troubleshooting to:
- Investigate alerts
- Validate incidents
- Rule out false positives
- Identify root causes
